    app端和H5页面交互规范

    转至元数据结尾
    • 创建： 应金平，最新修改于： 七月 05, 2018
    转至元数据起始

    后续和H5交互采用 JS bridge 方式进行交互，下面列出原生的方法

    JS调用注意点：

     调用方法时，必须通过window.bridge.xxx方法名，第一个参数为 要传递给app原生方法的参数, 第二个参数为 callback{Function}"，
    如果下列方法不需要传递参数，第一个方法参数可选，第二个参数必须要
    ,原因是IOS是通过callback的方式异步返回调用结果

    例如 

    window.bridge.updateShareButton("参数",function (param) { 
     doFunction(param)//param是Android和IOS原生方法返回参数，返回参数格式参考下面
    })
    或者 
    window.bridge.updateShareButton(function (param) { 
     doFunction(param)//param是Android和IOS原生方法返回参数，返回参数格式参考下面
    })

    参数解析：

    shareContent 分享内容

    shareImg 分享的图片

    shareTitle 分享的标题

    shareType 0 代表普通分享，包含标题，内容。 1代表纯图片分享 2代表小程序分享

    originalId 代表小程序分享原始ID，mediaPath 代表小程序分享路径，mediaImage 代表小程序分享缩略图，这三个只有shareType=2返回

    platForm 是包含分享到哪些平台，目前先暂时填写全平台，分类原因是便于以后扩展（比如我某H5页面只针对分享到 微信和朋友圈，另一个H5针对分享到微信朋友圈，微博,QQ），这一块还涉及到UI展示问题，需要具体需求，跟产品确定

         type: 1:微信 2 朋友圈 3 QQ 4QQ空间 5 微博 6 复制链接 

         shareUrl 当前平台分享的连接（因为每个平台可能分享的连接不同）

    3. closePage ： 关闭H5页面

                    // 点击 "刷新页面" 按钮
                    $(".closePage").on("click", function refreshPage() {

                            doFunction(param)//param为undefined
                        })
                    })

    5. getAppVersion： 获取app版本信息

     

    代码示例
    ?
    APP接收参数格式：
    {"actionType":"getAppVersion"}
    返回给JS的结果格式：
     x.x.x(字符串，例如 2.2.4)
     
    调用示例：           
            // 点击 "获取APP版本信息" 按钮
            $(".getAppVersion").on("click", function getAppVersion() {
                window.bridge.getAppVersion(function (param) {
                    doFunction(param)//param 为版本号值
                })
               
            })

    6. getAppToken： 获取app 登录token

     

    代码示例
    ?
    APP接收参数格式：
    {"actionType":"getAppToken"}
    返回给JS的结果格式：
     xxxx(字符串，例如 faf02dfoiiG7JK9LLD09Uuid98I===)
     
    调用示例：           
           // 点击 "获取token" 按钮
            $(".getAppToken").on("click", function getAppToken() {
                 window.bridge.getAppToken(function (param) {
                    doFunction(param)//param 为token值
                })
            })

    7. goAppLogin： 拉起APP登录，成功后回调，回调方法名appLoginCallback()，如果APP已经是登录状态，会直接执行回调方法appLoginCallback，回调方法appLoginCallback的参数result格式：

    ' {"result":true,"token":"faf4b0fd2c0e552a452509a97a9fa2ea"} ' (json字符串)

    经销商app：传递的

    ' {"result":true,"token":"faf4b0fd2c0e552a452509a97a9fa2ea","dealerId":"6877887"} ' (json字符串)

    必须在JS里声明回调方法，否则无法完成回调

    function appLoginCallback(result) {
     alert("appLoginCallback 执行,参数: " + result)
    }

     

    代码示例
    ?
    APP接收参数格式：
    {"actionType":"goAppLogin"}
    返回给JS的结果格式：
     无
     
    调用示例：           
           // 点击 "获取token" 按钮
            $(".goAppLogin").on("click", function goAppLogin() {
                 window.bridge.goAppLogin(function (param) {
                    doFunction(param)//param 为undefined
                })
            })

    8. openH5Screen： 打开一个原生 跳转到指定H5地址 的页面

     

    代码示例
    ?
    APP接收参数格式：
    {
        "actionType": "openH5Screen",
        "args": {
           "title": "我是H5标题",
            "url": "https://www.baidu.com/",
            "closeCurrenScreen": "true"
        }
    }
    返回给JS的结果格式：
    无
     
    调用示例：           
     // 点击 "打开H5链接新页面" 按钮
    $(".openH5Screen").on("click", function openH5Screen() {
        var result = window.bridge.openH5Screen({
            "title": "我是H5标题",
            "url": "https://www.baidu.com/",
            "closeCurrenScreen": "false"
        }, function (param) {
            doFunction(param)//param 为undefined
        })
    })
    })

    参数解析：

     

    title 为原生页面要显示的标题
    url 为H5地址链接
    closeCurrenScreen : 是否关闭原来调用的原生页面

     

    其他type待扩展
